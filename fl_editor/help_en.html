📖 FL Atlas – User Guide

1   Overview

FL Atlas is a graphical editor for the game data of Freelancer (2003, Digital Anvil). The program reads the INI files (systems, universe, solars, zones, etc.) and displays them on an interactive 2D map. Optionally, a 3D view can be enabled.

Standard Freelancer installations are supported, Freelancer HD Edition as well as mods like Crossfire.

2   Getting Started

  1. Enter the game path in the path input field at the top left (e.g. C:\Games\Freelancer) and press Enter or click Read.
  2. The system browser on the left shows all detected star systems. Click a system to load it into the 2D map.
  3. Alternatively, click 🌐 Universe in the toolbar to open the universe map with all systems.
💡   The game path is saved automatically and restored on next start.

6   Editing Objects & Zones

6.1   Objects

Select an object to edit it in the right editor panel. Here you can adjust position, rotation, archetype and many other fields. Changes are immediately reflected on the map.

Use the 🧊 3D Preview button to open a 3D preview model of the selected object (if the game directory is set correctly and the model exists).

6.2   Zones

Zones are shown as semi-transparent areas on the map. They can be shown/hidden with the Zones checkbox. Death and exclusion zones are highlighted in red and follow their parent object when moved.

6.3   Creating New Objects

In the right panel under Creation you can choose various creation types. The general workflow is:

  1. Click the desired button.
  2. Fill out the dialog and confirm.
  3. Click on the map to place the object.
  4. For zones and tradelanes: Second click or drag sets the size/end (rubber-band placement).
💡   During placement, the map shows a crosshair cursor and a yellow frame. Press Esc to cancel at any time.

Object

Creates a generic object. In the dialog you can choose nickname, archetype, loadout, and faction.

Sun

Creates a sun object. The dialog offers sun archetypes, radius, damage zone, star type, atmosphere range, and burn color.

Planet

Creates a planet. Similar to sun, but with planet archetypes and planet-specific parameters.

Jump Gate / Jump Hole

Creates a jump connection between two systems. In the dialog, choose the target system and type (gate or hole). For jump gates, an extra dialog appears for behavior, difficulty, loadout, pilot, and reputation.

Asteroid / Nebula

Creates a zone linked to an asteroid or nebula reference file. Type (asteroid/nebula), reference file, and zone name are set in the dialog. Placement uses two-click method (position + drag for size).

Zone

Creates a simple population zone without asteroid/nebula reference. In the dialog you can set name, comment, shape, and sort value. Two-click placement (position + rubber-band for size).

Tradelane

Creates a tradelane route. Start and end are set with two clicks (with rubber-band preview). In the dialog you can set ring count, spacing, and nickname prefix.

Base

Creates a complete base including all needed files: rooms, base INI, market files, universe.ini entry. The dialog offers fields for archetype, rooms, template, pilot, voice, head/body, price deviation, and IDS numbers.

7   Universe Map

The universe map shows all star systems and their connections (jump gates, jump holes, tradelanes).

  • Double click a system to open its system map.
  • Enable the Move checkbox to drag systems on the map.
  • Connection lines update automatically.
  • Click 💾 Save to write new positions to universe.ini.
  • Click ↩ Undo to undo all moves.
💡   The left panel shows the INI section of the currently selected system, so nickname, IDS references, and other metadata can be viewed and edited directly.

8   Creating & Editing Bases

8.1   Create New Base

Use the Base button in the creation area to create a complete base. The creation dialog offers:

  • Archetype and loadout of the station
  • Rooms (equipment, commodity, shipdealer, bar, etc.)
  • Template base as starting point
  • NPC config: pilot, voice, head, body
  • Price deviation and IDS numbers

After confirming, all files are generated automatically (rooms, base INI, market files) and the base is entered into universe.ini.

8.2   Edit Base

Select a dockable object on the map and click the Base button in the editing area. The edit dialog has four tabs:

TabContent
Properties Nickname, archetype, loadout, reputation, pilot, voice, head/body, IDS numbers, behavior, difficulty
Equipment Available equipment by category (weapons, shields & thrusters, misc, general). Tree view left, assigned entries right with level, rep, stock, price
Commodities Goods with base price from goods.ini, editable price multiplier and calculated final price
Ships Three ship slots with dropdown selection

9   Creating & Editing Tradelanes

9.1   New Tradelane

Click Tradelane in the creation area. Set start and end with two clicks (with rubber-band line). In the dialog you can set ring count, spacing, and nickname prefix.

9.2   Edit Tradelane

Use the Tradelane button in the editing area to select an existing tradelane chain, then delete or reposition it.

10   Edit Zone Population

Select a zone on the map and click Zone Population in the editing area. The dialog lets you edit:

  • Encounter parameters – Define which NPCs/enemies can appear in the zone
  • Faction assignments – Assign factions with weighting

New encounter parameter sections can be added directly.

11   System Settings

Use the ⚙️ [SYSTEM] – Settings button in the right panel to adjust the following system metadata:

  • Music – Space, danger, and battle tracks
  • Space color – Background color of space
  • Local faction – Dominant faction in the system
  • Ambient color – Ambient lighting
  • Dust – Particle effects in space
  • Backgrounds – Base stars, complex stars, and nebulae

12   Create New System

  1. Switch to universe view (🌐 Universe).
  2. Click 🌟 New System in the toolbar.
  3. Fill out the dialog: system name, nickname, sun type, nebula, music, etc.
  4. After confirming, a new folder with all needed INI files is created and the system is entered into universe.ini.
  5. Click a free spot on the universe map to place the system there.

13   IDS Management (Missing Names & Info)

Freelancer uses IDS numbers (ids_name, ids_info) to display object and zone names in-game. Entries with value 0 have no visible name or info in-game.

13.1   Step 1 – Scan for Missing IDS

  1. Switch to universe view.
  2. Click 🔍 Missing IDS in the toolbar.
  3. All system INI files are scanned. Two CSV files are created in the game directory:
FileColumns
missing_ids_name.csv System; Section; Nickname; Archetype; ids_name; givenname
missing_ids_info.csv System; Section; Nickname; Archetype; ids_info; xmlinfo

Open the CSV files in a spreadsheet editor (LibreOffice Calc, Excel) and enter the desired IDS number in the ids_name or ids_info column. The givenname and xmlinfo columns are for notes on the desired name or info description.

13.2   Step 2 – Apply IDS

  1. Click 📥 Apply IDS in the toolbar.
  2. All CSV rows with a filled ids_name or ids_info number are written to the respective system INI file.
  3. Successfully processed rows are removed from the CSV.
  4. When all rows are processed, the CSV file is deleted automatically.
💡   You can fill out the CSV step by step – only rows with a number are processed, the rest remain.

14   Saving & Safety

  • Before saving, a backup (.bak) of the original file is created automatically.
  • Unsaved changes are prompted when switching systems or exiting.
  • The save function writes all changes back in the same INI format expected by Freelancer.
⚠️   It is recommended to make a manual backup of game data before major changes.

16   File Formats

FL Atlas works with the INI files of the Freelancer data format. This format uses named sections in square brackets ([Object], [Zone], …) with key-value pairs. Duplicate keys within a section are supported.

Important files:

FileContent
universe.ini List of all systems, bases, and connections
*.ini (system folder) Objects, zones, nebulae, etc. of a single system
solararch.ini Archetype definitions for solar objects
market_misc.ini Equipment market data for a base
market_commodities.ini Commodity market data for a base
market_ships.ini Ship market data for a base
goods.ini All goods and equipment definitions with base prices
*.cmp / *.3db 3D models (compound / single-part)

17   Troubleshooting

  • No system is shown: Check if the game path points to the Freelancer root directory (contains DATA and EXE folders).
  • 3D view empty: Make sure Qt3D is available. On Linux, the package python3-pyside6.qt3d* may be needed.
  • Model not found: The archetype points to a model that does not exist in the game path. Check case sensitivity (Linux).
  • Saving fails: Check write permissions in the game directory.
  • CSV files not found (IDS import): Run 🔍 Missing IDS first to create the CSV files in the game directory.

18   Language & Themes

  • Language switcher: Use the DE/EN button in the toolbar to switch the entire interface language at any time. All texts, tooltips, and dialogs update instantly. Translations are stored in translations.json and can be customized externally.
  • Theme switcher: The 🎨 button lets you choose between four color schemes: Founder (dark blue), Dark (dark gray), Light (light), Custom (your own accent color). The selection is saved and restored on next start.
  • The help file is available in German and English and adapts to the selected language.
